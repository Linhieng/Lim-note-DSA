## 我的想法

初次编写代码时, 先考虑特殊情况/边界, 再考虑通用情况。
优化代码结构时, 先考虑通用情况(提取出核心代码), 再考虑特殊情况/边界。

多动笔画一画, 学会这一个技巧不仅仅有助于理解 DSA, 也能帮助你给别人讲解, 从而写出好的笔记。
